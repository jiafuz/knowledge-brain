# knowledge-ingest

将内容消化提炼后存入知识库。

**前置条件：**
1. 已完成 `/knowledge-init` 初始化分区
2. 先读取 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md) 了解认证

## 触发方式

- 用户说"消化这个"、"存入知识库"、"帮我消化一下"
- 用户直接给出链接（飞书文档、群消息、互联网文章）
- 用户粘贴文本内容

## 处理流程

详见 [ingest-flow.md](references/ingest-flow.md)

### 快速流程

```
识别来源 → 获取原文 → 提炼摘要 → 智能分类 → 创建文档 → 更新索引
```

## 来源识别与处理

| 来源 | 识别 | 处理 |
|------|------|------|
| 镜书文档 | `/docx/` `/doc/` `/wiki/` | `lark-cli docs +fetch` |
| 镜书群消息 | `/chat/` `oc_` | lark-im skill |
| 互联网文章 | 外部 URL | WebFetch |
| 粘贴文本 | 无 URL | 直接处理 |

## 智能分类规则

**主题判断：**
- 技术：编程、架构、API、工具、框架关键词
- 产品：需求、设计、用户、迭代关键词
- 管理：团队、流程、协作、决策关键词
- 其他：无法明确归类

**分区判断：**
- 内容是核心知识、重要参考 → 重点知识区
- 内容是一般文章、资讯 → 消化文章区

## 摘要格式

```markdown
## 背景
[内容背景简述]

## 核心观点
- [关键要点]

## 原文链接
[原始链接]

## 行动建议（可选）
[如有实践指导]

## 延伸思考（可选）
[关联知识]
```

## 需要的权限

- `wiki:node:create` - 创建知识库节点
- `docs:doc:read` - 读取文档
- `docs:doc:edit` - 创建文档
