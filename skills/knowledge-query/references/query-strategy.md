# 智能问答加载策略

## 判断流程

```
用户提问 → 分析语义 → 判断关联主题 → 检查索引 → 决定加载策略
```

## 加载策略

| 问题类型 | 特征 | 加载策略 |
|---------|------|---------|
| 明确指向某主题 | 提及具体主题词（技术架构、产品设计） | 加载该主题下所有文档 |
| 模糊问题 | 通用关键词（怎么解决、如何处理） | 加载最相关 1-3 篇文档（关键词匹配） |
| 与知识库无关 | 不涉及知识库主题 | 不加载，用已有知识回答 |

## 主题关键词映射

| 主题 | 关键词 |
|------|--------|
| 技术 | 编程、代码、架构、API、框架、bug、性能、部署 |
| 产品 | 需求、设计、用户、迭代、功能、体验、原型 |
| 管理 | 团队、流程、协作、决策、会议、任务、目标 |

## 回退机制

当知识库索引不存在或无相关文档时：
- 直接用 Claude 已有知识回答
- 回答中标注："以上基于我已有知识，知识库中暂无相关内容"

## 按需加载原文

从索引获取文档 token 后：
```bash
lark-cli docs +fetch --doc "<token>" --format pretty
```

加载全文到对话上下文，结合知识库内容和已有知识回答。
