# knowledge-brain - 个人第二大脑

> 智能管理你的飞书知识库，让 Claude 成为你的知识助手

## 功能特性

- **智能消化** - 扔给我任何内容（飞书文档、群消息、网页链接、粘贴文本），自动提炼摘要存入知识库
- **自动分类** - 根据内容智能判断主题（技术/产品/管理等），无需手动归类
- **预加载知识** - 会话启动时自动预加载索引，快速响应你的问题
- **智能问答** - 根据问题语义按需加载相关知识库内容回答

## 快速开始

### 1. 初始化知识库分区

首次使用需要初始化分区结构：

```
/knowledge-init
```

这会在你的飞书知识库中创建两个分区：
- **重点知识区** - 存放核心知识、重要参考
- **消化文章区** - 存放消化后的文章摘要

### 2. 消化内容

把任何内容扔给我，我会帮你消化：

```
/digest https://example.com/article
/digest 飞书文档链接
/digest 粘贴的文本内容...
```

支持的来源类型：
| 来源 | 示例 |
|------|------|
| 飞书文档 | `https://xxx.feishu.cn/docx/xxx` |
| 飞书 Wiki | `https://xxx.feishu.cn/wiki/xxx` |
| 飞书群消息 | `https://xxx.feishu.cn/chat/xxx` |
| 互联网文章 | `https://blog.example.com/xxx` |
| 粘贴文本 | 直接粘贴内容 |

### 3. 查询知识

直接问我问题，我会自动从知识库中查找相关内容：

```
知识库里有哪些 Git 相关的内容？
怎么解决代码合并冲突？
```

### 4. 同步知识库

如果你在飞书中直接修改了知识库，可以同步更新索引：

```
/sync
```

### 5. 查看状态

查看知识库当前状态：

```
/knowledge-status
```

## 知识库结构

```
个人知识库
├── 重点知识区          # 核心知识，会预加载到"脑海"
│   ├── 技术类文档
│   ├── 产品类文档
│   └── ...
└── 消化文章区          # 消化后的摘要，帮你判断是否值得读原文
    ├── 技术类文章
    ├── 产品类文章
    └── ...
```

## 摘要格式

每篇消化后的文章都会生成结构化摘要：

```markdown
## 背景
[内容背景简述]

## 核心观点
- [关键要点 1]
- [关键要点 2]
- ...

## 原文链接
[原始链接/来源]

## 行动建议（可选）
[如有实践指导]

## 延伸思考（可选）
[关联知识、后续待深入了解的点]
```

## 工作原理

### 预加载机制

1. **会话启动时**：自动加载索引和重点知识区摘要（轻量）
2. **提问时**：智能判断问题关联，按需加载原文
3. **手动触发**：`/sync` 深度学习重点知识区全文

### 智能分类

根据内容关键词自动判断主题：
- **技术**：编程、架构、API、框架、bug、性能、部署
- **产品**：需求、设计、用户、迭代、功能、体验
- **管理**：团队、流程、协作、决策、会议、任务
- **其他**：无法明确归类的内容

## 常见问题

### Q: 如何区分重点知识和消化文章？

**重点知识区**：
- 需要经常查阅的核心知识
- 重要的参考资料
- 希望我"记住"的内容

**消化文章区**：
- 一般性的文章、资讯
- 帮你快速判断是否值得读原文
- 知识密度可能较低的内容

### Q: 知识库内容修改后如何更新？

执行 `/sync` 命令会全量扫描知识库并更新索引。

### Q: 如何删除知识库中的文档？

目前需要手动在飞书中删除，删除后执行 `/sync` 更新索引。

## 技巧

1. **批量消化**：可以一次给我多个链接，我会逐个处理
2. **指定主题**：如果自动分类不准确，可以在内容中加上 `#技术` 或 `#产品` 等标签
3. **定期同步**：建议每周执行一次 `/sync` 保持索引最新
